### 2. Momentum (метод моментов)
- Добавлена инерция: `v := βv - η∇f(w)`, `w := w + v`
- Параметры: `η = 0.05`, `β = 0.03`, 3000 эпох
- Инерция применяется к весам и свободному члену

### 3. Nesterov Accelerated Gradient (NAG)
- Градиент вычисляется в "заглядывающей вперёд" точке: `∇f(w + βv)`
- Параметры: `η = 0.05`, `β = 0.3`, 300 эпох
- Метод Нестерова, сходимость O(1/k²)

### 4. Метод эллипсоидов
- Вместо обновления весов — сжатие 21-мерного эллипсоида
- Формулы: `h = Ag/√(gᵀAg)`, `c := c - h/(n+1)`, обновление матрицы A
- Параметры: `R = 400`, 150 эпох

## \[1. Описание зоны ответственности и реализованные методы\]

### 1.1. Стохастические градиентные методы
* Разработка алгоритмов: реализация оптимизаторов первого порядка AdaGrad и AdamW выполнена «с нуля» с использованием библиотеки numpy.
* Масштабирование и инференс: интегрированы механизмы масштабирования входных признаков и обратного преобразования предсказаний модели для расчета метрик качества на исходной шкале цен.

### 1.2. Методология борьбы с переобучением
* Валидация: Для предотвращения утечки данных (*data leakage*) при настройке гиперпараметров реализована схема разделения выборки на три независимых компонента:
* *Train* - расчет и обновление весовых коэффициентов модели.
* *Validation* - оптимизация коэффициента регуляризации $\lambda$ и фиксация критерия ранней остановки (*Early Stopping*).
* *Test* - изолированная проверка финальной обобщающей способности модели.

* Регуляризация:
* Добавлен штраф $L1$ (Lasso) для обеспечения разреженности векторов параметров:
$$J_{L1} = \frac{1}{B} \sum_{i=1}^B (y_i - \hat{y}_i)^2 + \lambda \sum_{j=1}^M \vert{}w_j\vert{}$$

* Добавлен штраф $L2$ (Ridge) для контроля амплитуды весовых коэффициентов:
$$J_{L2} = \frac{1}{B} \sum_{i=1}^B (y_i - \hat{y}_i)^2 + \lambda \sum_{j=1}^M w_j^2$$

### 1.3. Online Learning
* Создана симуляция непрерывного поступления данных для обучения моделей в режиме реального времени.
* Реализованы алгоритмы FTRL (Follow the Regularized Leader) и Adaptive FTRL, адаптированные под динамический поток.

## \[2. Результаты исследования градиентных методов (ResearchFinal01)\]
* Специфика AdamW: Экспериментально подтверждена эффективность механизма Decoupled Weight Decay. Штраф за L2-норму применяется напрямую к весам на этапе обновления, что изолирует адаптивный шаг обучения от влияния градиентов регуляризатора.

## \[3. Сравнение адаптивных градиентных методов с онлайн-методами (OnlineLearning.ipynb)\]
На основе экспериментов в режиме обработки непрерывного потока данных зафиксированы следующие различия:

### 3.1. Поведение при дефиците данных
* Адаптивные SGD-методы: На начальных этапах (объем выборки до 1000 объектов) алгоритмы AdaGrad и AdamW демонстрируют критическую нестабильность. Значение метрики $R^2$ для AdamW снижалось до $-694.0844$, для AdaGrad - до $-42.2243$.
* Онлайн-методы: Adaptive FTRL в идентичных условиях функционирует стабильно, фиксируя показатель $R^2 = 0.7228$ на отметке 1000 объектов.
* Причина различий: показатель разреженности (*Sparsity*) у AdaGrad и AdamW стабильно равен 0, так как они не имеют встроенного механизма жесткого зануления весов. Модели накапливают исторический градиентный шум и начинают конкурировать с онлайн-методами по точности только при накоплении выборки объемом от 5000 объектов.

### 3.2. Устойчивость к Concept Drift
* Реакция на сдвиг: в районе 2200-го шага потока зафиксировано резкое, вероятно, изменение структуры распределения данных.
* Адаптация FTRL: онлайн-модели семейства FTRL временно снизили долю нулевых весов, активировав необходимые коэффициенты под новые параметры выборки, и восстановили качество предсказания до уровня $R^2 \approx 0.75$.
* Адаптация SGD: Алгоритмы AdaGrad и AdamW адаптировались к сдвигу значительно медленнее, восстановив итоговую метрику лишь до уровня $R^2 \approx 0.42$.

### 3.3. Динамика шага в Adaptive FTRL
* Проблема быстрого затухания: при обновлении мини-батчами базовый алгоритм Adaptive FTRL показал избыточно высокую скорость затухания шага обучения, останавливаясь в субоптимальной точке.
* Решение: путем калибровки параметров затухания скорости обучения до значений $\alpha = 1.0$ и $\beta = 1.0$ была достигнута стабилизация модели. Адаптивная версия сравнялась по точности с классическим FTRL, продемонстрировав более плавную сходимость на интервале от 3000 до 5000 объектов.

## 4. Реализация стохастического градиентного спуска (SGD)

* **Борьба с шумом:** Для стабилизации траектории сходимости реализован механизм динамического уменьшения шага обучения (*learning rate decay*), снизивший скорость обучения в процессе 30 эпох с `0.01000` до `0.00408`.
* **Оценка эффективности:** На тестовой выборке метод показал высокую скорость вычислений при `MSE: 0.04730`, `RMSE: 0.21748` и `R²: 0.81912`. Анализ истории ошибки выявил, что минимальный MSE `0.04531` был достигнут на 20-й эпохе, после чего модель колебалась около этого значения, что подтверждает классическое поведение данного алгоритма.
