Использована функция градиента, разработанная в рамках проекта, на основе которой реализовано итеративное обновление весовых коэффициентов модели.
В процессе реализации:
* выполнена настройка коэффициента скорости обучения (learning rate);

Для предотвращения переобучения реализованы два наиболее распространённых метода регуляризации.
* Добавлена L2-регуляризация, ограничивающая величину весовых коэффициентов модели за счёт штрафа на квадрат их нормы.

## 3. Реализация адаптивного метода Adam
Реализован алгоритм адаптивной оптимизации Adam.
В отличие от классического градиентного спуска, данный метод автоматически изменяет величину шага обучения для каждого параметра модели на основе оценок первого и второго моментов градиента.

* L1-регуляризация.

### 2. Momentum (метод моментов)
- Добавлена инерция: `v := βv - η∇f(w)`, `w := w + v`
- Параметры: `η = 0.05`, `β = 0.03`, 3000 эпох
- Инерция применяется к весам и свободному члену

### 3. Nesterov Accelerated Gradient (NAG)
- Градиент вычисляется в "заглядывающей вперёд" точке: `∇f(w + βv)`
- Параметры: `η = 0.05`, `β = 0.3`, 300 эпох
- Метод Нестерова, сходимость O(1/k²)

### 4. Метод эллипсоидов
- Вместо обновления весов — сжатие 21-мерного эллипсоида
- Формулы: `h = Ag/√(gᵀAg)`, `c := c - h/(n+1)`, обновление матрицы A
- Параметры: `R = 400`, 150 эпох

## Сравнение адаптивных градиентных методов с онлайн-методами (OnlineLearning.ipynb)
На основе вычислительных экспериментов в условиях симуляции непрерывного потока данных зафиксированы следующие различия:
### Поведение при дефиците данных
* Адаптивные методы (AdaGrad, AdamW): На начальных этапах обработки потока (объем выборки до 1000 объектов) алгоритмы демонстрируют критическую нестабильность, фиксируя крайне низкие показатели качества (значение метрики R^2 для AdamW снижалось до -694.0844, для AdaGrad - до -42.2243).
* Онлайн-методы: Алгоритм Adaptive FTRL в идентичных условиях функционирует стабильно, показывая R^2 = 0.7228 на отметке 1000 объектов.
* Причина различий: Показатель разреженности (Sparsity) у AdaGrad и AdamW стабильно равен 0, так как у них отсутствует встроенный механизм жесткого зануления весов. Модели накапливают исторический градиентный шум и начинают конкурировать с онлайн-методами по точности только при накоплении выборки объемом от 5000 объектов.
### Устойчивость к Concept Drift
* Реакция на сдвиг: около 2200 шага потока (для выборки «all») зафиксировано резкое изменение структуры распределения данных (concept drift), вызвавшее падение точности у всех тестируемых моделей.
* Адаптация FTRL: онлайн-модели семейства FTRL продемонстрировали высокую скорость адаптации к сдвигу. За счет временного снижения доли нулевых весов (активации необходимых коэффициентов под изменившуюся структуру данных) они быстро переобучились и восстановили качество предсказания до уровня R^2 ~ 0.75.
* Адаптация SGD: алгоритмы AdaGrad и AdamW подстроились под новые данные медленнее, восстановив итоговую метрику лишь до уровня R^2 \approx 0.42.
### Динамика шага в Adaptive FTRL
* Проблема преждевременного затухания: При обновлении мини-батчами базовый вариант Adaptive FTRL характеризовался избыточно высокой скоростью затухания шага обучения, останавливаясь в субоптимальной точке с неудовлетворительными метриками качества.
* Решение: Проблема устранена путем ручной настройки параметров затухания скорости обучения до значений alpha = 1.0 и beta = 1.0. Оптимизированный адаптивный вариант сравнялся по точности с классическим FTRL, продемонстрировав более плавную и стабильную траекторию сходимости на интервале от 3000 до 5000 объектов.

## 2. Реализация точного аналитического решения (МНК)
Для фиксации предела точности линейной регрессии на наших данных был реализован метод наименьших квадратов (МНК) через нормальное уравнение:
$$w = (X^T X)^{-1} X^T y$$

В процессе реализации:
* Проведена оценка качества точного решения на финальном тестовом датасете из 8 000 объектов, зафиксировавшая эталонные метрики: `MSE: 0.04248`, `RMSE: 0.20610`, `R²: 0.83755`.

## 3. Исследование масштабируемости классического пакетного спуска (Batch GD)
Проведен подробный сравнительный анализ классического градиентного спуска при обучении на подвыборках разного объема (1000, 3000 и 5000 объектов) с тестированием на общем массиве из 8 000 квартир:
* **На 1000 объектах:** модель продемонстрировала качество `MSE: 0.04290`, `RMSE: 0.20713`, `R²: 0.83592`.
* **На 3000 объектах:** метрики улучшились до `MSE: 0.04258`, `RMSE: 0.20635`, `R²: 0.83716`.
* **На 5000 объектах:** итерационный алгоритм сошелся вплотную к теоретическому максимуму МНК, выдав идентичный результат до пятого знака после запятой (`MSE: 0.04248`, `RMSE: 0.20610`, `R²: 0.83755`).

## 4. Реализация стохастического градиентного спуска (SGD)
С нуля реализован алгоритм стохастического градиентного спуска (SGD), вычисляющий градиент функции потерь и обновляющий веса по одному случайному объекту на каждом шаге.
* **Борьба с шумом:** Для стабилизации траектории сходимости и компенсации стохастических колебаний внедрен механизм динамического затухания шага обучения (*learning rate decay*), снизивший скорость обучения в процессе 30 эпох с `0.01000` до `0.00408`.
* **Оценка эффективности:** На тестовой выборке метод показал высокую скорость вычислений при качестве `MSE: 0.04730`, `RMSE: 0.21748` и `R²: 0.81912`. Анализ истории лосса выявил, что минимальный MSE на обучении `0.04531` был достигнут на 20-й эпохе, после чего модель колебалась около этого значения, что подтверждает классическое поведение стохастических оптимизаторов.
